Element-UI Related

As previously described, we used Element-UI as the component library for the front-end in this project. While component libraries offer convenience, they also have some limitations and problems.\

Version

For example, the versioning of Element-UI is an issue. There are two versions of Element-UI, the native Element-UI and Element-Plus. there is not much difference between the two, in terms of style Element-Plus has more compatibility, but in terms of stability this new release is not as good as Element-UI. therefore, when creating the project at the beginning, I I chose Element-UI because it is compatible with the Vue 2.0 version we are using. However, one of the team members was using Vue 3.0 and therefore Element-Plus, which led to a lot of conflicts when we tried to merge our branches.

Style

There is also the challenge of modifying the style of these provided components. This is because these components are highly encapsulated and do not provide much of an interface for users to modify in order to maintain consistency of style. As an example, when trying to change the background colour of some components, it was found that modifying the CSS statement directly did not work and have to be use the keyword !important. Another example is when using the checkBox component, where the default size does not meet the requirements. However, the size of this component is fixed according to the size of the window and changing its size directly does not work. The final solution was to use CSS statements to scale it up.

background-color: #f5f5f5 !important;

transform: scale(1.8);
